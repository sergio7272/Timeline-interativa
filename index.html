Git e GitHub

2005

Linus Torvalds cria o Git

Marco

Após conflitos com o BitKeeper, Torvalds desenvolve o Git em apenas 10 dias para gerenciar o kernel Linux — nascendo o sistema de controle de versão distribuído mais usado no mundo.

2008

Fundação do GitHub

Contexto

GitHub transforma o Git numa plataforma social de código, permitindo pull requests, issues e colaboração em massa — democratizando o open source.

2018

Microsoft adquire o GitHub

Marco

Aquisição de US$ 7,5 bilhões consolida o GitHub como infraestrutura essencial. Repos privados gratuitos e GitHub Actions transformam o fluxo de CI/CD.

2024

Monorepos e escala global

Desafio

Gerenciar repositórios com milhões de commits, milhares de contribuidores e pipelines complexos exige novas estratégias de branching, segurança e governança.

UML

1997

UML 1.0 adotada pela OMG

Marco

Booch, Rumbaugh e Jacobson unificam notações de modelagem OO. A UML torna-se o padrão para documentar arquiteturas de software em grandes organizações.

2005

UML 2.0 e diagramas avançados

Contexto

Introdução de 13 tipos de diagrama, incluindo sequência, atividades e estados. Ferramentas como Enterprise Architect e Visual Paradigm popularizam a modelagem visual.

2015

Agilidade vs. documentação pesada

Contexto

Métodos ágeis questionam a UML formal. Surge o "diagrams as code" (PlantUML, Mermaid) — modelagem leve integrada ao repositório e ao fluxo de desenvolvimento.

2024

Modelar sistemas distribuídos

Desafio

Microsserviços, event-driven e serverless exigem novos padrões. O C4 Model e Architecture Decision Records (ADR) complementam a UML para capturar decisões arquiteturais.
